Inventory Manager CLI

Цель:
Разработать CLI-приложение на Python, PostgreSQL и SQLAlchemy ORM.
Приложение должно позволять вести складской учёт: добавлять товары, категории, поставщиков, складские операции и смотреть отчёты по остаткам.

Необходимо:
- создать новую базу данных PostgreSQL
- настроить подключение через .env
- описать новые ORM-модели
- создать таблицы через SQLAlchemy
- реализовать CRUD-операции
- реализовать JOIN-запросы
- реализовать агрегатные отчёты
- обработать ошибки базы данных через IntegrityError и rollback
- оформить проект так, чтобы его можно было выложить на GitHub

Тема проекта:
Inventory Manager CLI — складской учёт.

Описание:
На складе хранятся товары.
Каждый товар относится к категории.
У товара может быть поставщик.
По товарам происходят складские операции:
- поступление товара на склад
- списание товара со склада
- корректировка количества

Приложение должно помогать пользователю видеть:
- какие товары есть на складе
- сколько единиц каждого товара осталось
- какие товары заканчиваются
- какие товары привязаны к конкретному поставщику
- какие операции происходили на складе
- общую стоимость остатков

Основные сущности:

1. categories
Категории товаров.

Примеры:
- Electronics
- Furniture
- Office Supplies
- Tools
- Food

Поля:
- id
- name

Ограничения:
- название категории не может быть пустым
- название категории должно быть уникальным

Связи:
- одна категория может иметь много товаров


2. suppliers
Поставщики товаров.

Примеры:
- TechTrade
- OfficeMarket
- WoodFactory
- GlobalTools

Поля:
- id
- name
- phone
- email
- is_active
- created_at

Ограничения:
- название поставщика не может быть пустым
- название поставщика должно быть уникальным
- email должен быть уникальным, если он указан
- is_active по умолчанию True
- created_at по умолчанию текущая дата

Связи:
- один поставщик может поставлять много товаров


3. products
Товары на складе.

Примеры:
- Laptop Lenovo ThinkPad
- Office Chair
- A4 Paper Pack
- Screwdriver Set
- Wireless Mouse

Поля:
- id
- name
- sku
- category_id
- supplier_id
- purchase_price
- selling_price
- min_quantity
- is_active
- created_at

Ограничения:
- название товара не может быть пустым
- sku должен быть уникальным
- purchase_price должен быть больше или равен 0
- selling_price должен быть больше или равен 0
- min_quantity должен быть больше или равен 0
- is_active по умолчанию True
- created_at по умолчанию текущая дата

Связи:
- товар принадлежит одной категории
- товар может иметь одного поставщика
- один товар может иметь много складских операций


4. stock_movements
Складские операции.

Типы операций:
- IN       — поступление товара
- OUT      — списание товара
- ADJUST   — корректировка остатка

Поля:
- id
- product_id
- movement_type
- quantity
- comment
- created_at

Ограничения:
- product_id обязателен
- movement_type может быть только IN, OUT или ADJUST
- quantity должен быть больше 0
- created_at по умолчанию текущая дата

Связи:
- каждая операция относится к одному товару


Связи между таблицами:

categories 1 -> many products
suppliers  1 -> many products
products   1 -> many stock_movements

CRUD-задания:

1. Создание данных

Нужно реализовать:
- создать категорию
- создать поставщика
- создать товар
- добавить складскую операцию

Примеры сценариев:
- создать категорию Electronics
- создать поставщика TechTrade
- создать товар Laptop Lenovo ThinkPad
- добавить поступление 10 штук
- добавить списание 2 штук
- посмотреть текущий остаток


2. Чтение данных

Нужно реализовать:
- получить все категории
- получить всех поставщиков
- получить все товары
- получить товар по id
- получить товары конкретной категории
- получить товары конкретного поставщика
- получить все складские операции
- получить операции по конкретному товару


3. Обновление данных

Нужно реализовать минимум:
- изменить название категории
- изменить контактные данные поставщика
- изменить закупочную цену товара
- изменить продажную цену товара
- изменить минимальный остаток товара
- деактивировать товар
- деактивировать поставщика

Важно:
Деактивация не удаляет запись из базы.
Она просто меняет is_active на False.


4. Удаление данных

Нужно реализовать:
- удалить категорию
- удалить поставщика
- удалить товар
- удалить складскую операцию


Примеры:
- нельзя удалить категорию, если в ней есть товары
- нельзя удалить поставщика, если к нему привязаны товары
- нельзя удалить товар, если по нему есть складские операции

В таких случаях приложение должно выводить понятное сообщение:
"Нельзя удалить товар, потому что по нему уже есть складские операции."


Складские операции:

Тип IN:
Означает поступление товара на склад.

Пример:
Товар: Laptop Lenovo ThinkPad
Количество: 10
Тип операции: IN

Тип OUT:
Означает списание товара со склада.

Пример:
Товар: Laptop Lenovo ThinkPad
Количество: 2
Тип операции: OUT

Тип ADJUST:
Означает ручную корректировку остатка.

Пример:
Товар: Laptop Lenovo ThinkPad
Количество: 1
Тип операции: ADJUST

Для базовой версии проекта можно считать:
- IN увеличивает остаток
- OUT уменьшает остаток
- ADJUST увеличивает остаток

Расчёт текущего остатка:

Остаток товара считается не отдельным полем, а на основе операций.

Формула:

остаток = сумма IN + сумма ADJUST - сумма OUT

Пример:

IN 10
OUT 2
ADJUST 1

Текущий остаток:
10 - 2 + 1 = 9


Обязательные JOIN-запросы:

1. Товары с названиями категорий и поставщиков

Вывести:

ID | Товар | SKU | Категория | Поставщик | Закупочная цена | Продажная цена | Активен


2. История операций с названием товара

Вывести:

Дата | Товар | SKU | Тип операции | Количество | Комментарий


3. Товары конкретной категории

Пользователь вводит id категории.
Приложение показывает все товары этой категории.


4. Товары конкретного поставщика

Пользователь вводит id поставщика.
Приложение показывает все товары этого поставщика.


5. Операции по конкретному товару

Пользователь вводит id товара.
Приложение показывает историю операций по этому товару.


Обязательные агрегатные отчёты:

1. Количество товаров в каждой категории

Пример вывода:

Electronics: 12
Furniture: 5
Office Supplies: 23


2. Количество товаров у каждого поставщика

Пример вывода:

TechTrade: 8
OfficeMarket: 15
WoodFactory: 4


3. Текущий остаток по каждому товару

Пример вывода:

Laptop Lenovo ThinkPad: 8
Office Chair: 15
A4 Paper Pack: 120


4. Товары, которые заканчиваются

Нужно вывести товары, у которых текущий остаток меньше или равен min_quantity.

Пример:

Wireless Mouse
Остаток: 3
Минимальный остаток: 5

A4 Paper Pack
Остаток: 8
Минимальный остаток: 10


5. Общая закупочная стоимость склада

Формула:

текущий остаток товара * purchase_price

Пример:

Laptop Lenovo ThinkPad:
Остаток: 8
Закупочная цена: 70000
Стоимость остатка: 560000

Итого по складу: 560000


6. Потенциальная выручка склада

Формула:

текущий остаток товара * selling_price

Пример:

Laptop Lenovo ThinkPad:
Остаток: 8
Продажная цена: 95000
Потенциальная выручка: 760000


7. Потенциальная прибыль склада

Формула:

текущий остаток товара * (selling_price - purchase_price)

Пример:

Laptop Lenovo ThinkPad:
Остаток: 8
Закупочная цена: 70000
Продажная цена: 95000
Потенциальная прибыль: 200000


Обязательная обработка ошибок:

Необходимо обработать:

1. Создание двух категорий с одинаковым названием
2. Создание двух поставщиков с одинаковым названием
3. Создание двух товаров с одинаковым sku
4. Создание товара с отрицательной закупочной ценой
5. Создание товара с отрицательной продажной ценой
6. Создание товара с отрицательным минимальным остатком
7. Создание складской операции с quantity <= 0
8. Создание складской операции с неправильным movement_type
9. Создание товара с несуществующим category_id
10. Создание товара с несуществующим supplier_id
11. Создание операции с несуществующим product_id
12. Удаление категории, у которой есть товары
13. Удаление поставщика, у которого есть товары
14. Удаление товара, у которого есть складские операции

Во всех случаях нужно использовать:
- try
- except IntegrityError
- session.rollback()
- понятное сообщение для пользователя
